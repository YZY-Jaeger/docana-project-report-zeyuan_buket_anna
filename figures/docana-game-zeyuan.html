Preprocessing is an important step in our analysis. We remove elements that are not helpful for our task, such as links, stop words, and anything that is not a letter or space. In addition to stop words, we noticed that some phrases still appeared as important despite the stop word removal for example, phrases like quick play, ranked games, or playing overwatch. To address this, we also removed certain domain-specific words like play, playing, game, etc., so that phrases containing these words would not dominate the results. Furthermore, we removed any text that appears after a TL;DR (in any form) because it could bias the phrase extraction towards words that are common in summaries. Summaries often repeat key points in a condensed way rather than reflecting the natural flow of the document.

import re
import nltk
from nltk.corpus import stopwords

nltk.download('stopwords')
stop_words = set(stopwords.words('english'))

common_words = {"game", "games", "gaming", "player", "play", "playing"}

def remove_tldr(text):
    pattern = re.compile(r'(tl;dr|tldr|tl dr|tl:dr)', re.IGNORECASE)
    match = pattern.search(text)
    if match:
        return text[:match.start()].strip()
    return text
    
def preprocess(text):
    text = remove_tldr(text)
    text = text.lower()
    text = re.sub(r"http\S+|www\S+|https\S+", '', text) #removes links if any
    text = re.sub(r'[^a-z\s]', ' ', text) #replaces anything that is not a letter or space with a space
    text = re.sub(r'\s+', ' ', text).strip() #replace multiple whitespace characters with a single space
    tokens = text.split()
    tokens = [word for word in tokens if word not in stop_words and word not in common_words]
    return " ".join(tokens)

# Apply to your filtered gaming dataset
df_games['cleaned_body'] = df_games['body'].apply(preprocess)

Training Word2Vec

The Word2Vec model was trained on this phrased corpus (corpus_phrased), where common multi-word expressions were merged with underscores.

We used the skip-gram architecture to learn embeddings that predict surrounding context words.

The model had an embedding size of 100 dimensions, a window size of 5, and ignored tokens that appeared fewer than 5 times.

In[]:
from gensim.models import Word2Vec
from gensim.models.callbacks import CallbackAny2Vec
from tqdm import tqdm

class EpochLogger(CallbackAny2Vec):
    def __init__(self, epochs):
        self.epochs = epochs
        self.pbar = tqdm(total=epochs, desc='Training Epochs')
    
    def on_epoch_end(self, model):
        self.pbar.update(1)
    
    def on_train_end(self, model):
        self.pbar.close()

# Then train the model with the callback

epoch_logger = EpochLogger(epochs=5)

w2v_model = Word2Vec(
    sentences=corpus_phrased,
    vector_size=100,   # Embedding dimensions
    window=5,    # Context window
    min_count=5,  # Ignore rare words
    sg=1,    # Use skip-gram (1) or CBOW (0)
    workers=4,
    epochs=5, # increase this for better training (epochs = 10)
    callbacks=[epoch_logger]
)

For each subreddit, we identified the top 20 most distinctive phrases (those with the highest TF-IDF scores). These phrases were collected into a set, merged with underscores, and converted to the same format (e.g., legend of zelda - legend_of_zelda), since the Word2Vec model was trained on text where frequent phrases were joined with underscores.

PCA

Since these embeddings are high-dimensional, we applied Principal Component Analysis (PCA) to reduce them to 2 dimensions (X_reduced) for visualization.

In[45]:
import numpy as np
from sklearn.decomposition import PCA
import matplotlib.pyplot as plt

# Filter to only those in the vocab
valid_phrases = [p for p in unique_phrases if p in w2v_model.wv]

# Get their vectors
X = np.array([w2v_model.wv[p] for p in valid_phrases])

# Reduce to 2D
pca = PCA(n_components=2)
X_reduced = pca.fit_transform(X)

# Plot
plt.figure(figsize=(12, 8))
plt.scatter(X_reduced[:, 0], X_reduced[:, 1])

for i, phrase in enumerate(valid_phrases):
    plt.annotate(phrase, (X_reduced[i, 0], X_reduced[i, 1]))

plt.title("2D PCA Visualization of Subreddit Bigrams")
plt.xlabel("PCA 1")
plt.ylabel("PCA 2")
plt.grid(True)
plt.show()

Phrases that appear close together likely have similar meanings or occur in similar contexts in subreddit data.

Phrases that are far apart are less related in terms of usage/context.

Since TfidfVectorizer was set to ngram_range=(2, 3), it only considered bigrams and trigrams (no unigrams). As a result, all the top terms I extracted were bigrams or trigrams in PCA.

K-Means Clustering

KMeans groups phrases with similar vector representations.

-The model tries to group together bigrams/trigrams that appeared in similar contexts across the data. For example, phrases about a particular game or character might cluster together because their usage patterns are similar.

-Each cluster may represent a different topic (e.g., one cluster's tokens could be related to Overwatch, another to Pokmon, etc.).
